# 개요

### 웹 브라우저

```.html``` 이라는 확장자를 가진 파일을 구동하는 것이 웹 브라우저

**웹 브라우저의 예**  
크롬, 엣지, 익스플로러, 사파리, 파이어폭스 ..

**웹 브라우저의 역할**
 - html 파일의 뷰어 역할 
 - 사용자의 웹 서핑을 위한 다양한 부가 기능 ( 즐겨찾기, 방문 기록 저장 ) 제공
 - 개발자에게 필요한 웹 페이지 분석을 위한 개발 도구도 제공

### HTML  

**H**yper **T**ext **M**arkup **L**anguage

**Hyper Text**  
다른 위치나 다른 웹 페이지로 이동할 수 있도록 만들어진 요소들  
지금은 웹 문서를 이루고 있는 요소 하나 하나가 하이퍼 텍스트를 의미

**Markup Language**  
데이터를 그 자체로 어떤 방식으로 어디에 어떻게 표현할지 보여질 지를 정의하는 언어  

**Programming Language**  
데이터를 가지고 가공을 하고 '어떤 것을 해라'는 명령을 내리는 언어

### 프론트엔드를 구성하는 언어

  언어    |                 역할
----------|------------------------------------
html      | 웹의 뼈대를 만드는 언어, 요소를 정의
css       | 각 요소들을 색깔, 모양을 변경할 수 있고 위치나 여러가지를 바꿈
javascript| 동적인 요소를 담당

### 웹의 특성

**표준**  
html문서를 만들었는데 3가지 브라우저에서 열었을 때 다 다르게 보여지는데 버튼이라는 특성마저 다르게 만들어진다면 복잡하게 될것이다.  
그래서 표준을 만들어서 HTML 5로 규격화 하였다.
이 표준을 준수하여 만들면 모든 브라우저에서 의도한 대로 보여질 수 있다.

**접근성**  
모든 사용자들이 브라우저를 통해 쉽게 사용할 수 있도록 하는 것  

예시) 크롬  
검색어를 입력할 때 키보드를 통해서 tab버튼으로 다른 요소로 접근이 가능하게 하는 것,  
키보드가 없다면 마우스로 입력할 수 있도록 하는 것,  
이런 것을 접근성을 높이는 방법이라고 할 수 있다.

**호환성 : 크로스 브라우징**  
웹 브라우저의 버전, 종류에 관계없는 접근이 가능하도록 하는 것

### VSCode

확장 프로그램
 - auto rename tag : 태그 작성 시에 여는 태그와 닫는 태그를 동시에 수정이 가능하게 해줌
 - live server : 코드를 수정하고 저장을 하면 브라우저에 실시간으로 반영되어 보여줌
 - prettier : 문법에 맞게 자동으로 코드를 수정 보완해줌
